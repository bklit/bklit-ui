Studio UI is the design system for the **chart editor** — sidebars, property rows, motion controls, and pickers. Chart rendering primitives live in `@bklitui/ui` (see the main site docs); **this app** documents editor-only UI in `packages/studio`.

Local-only — not deployed.

## Quick links

| Resource | URL |
|----------|-----|
| This docs app | http://localhost:3003/docs |
| Full editor | http://localhost:3000/studio |
| Add shadcn components | `cd packages/studio && pnpm ui:add <name>` |

## Architecture

```
packages/studio/
├── src/ui/              # shadcn primitives (components.json → pnpm ui:add)
├── src/components/controls/  # chart property widgets
├── src/lib/studio-chrome-classes.ts  # shared Tailwind for sidebar chrome
├── src/styles/
│   ├── studio-theme.css      # scoped .studio-shell chart + shadcn vars
│   └── studio-chrome.css     # WebKit scrollbar rules
└── components.json      # shadcn CLI config (ui alias → @/ui)

packages/icons/          # @bklitui/icons — Central Icon System wrapper
```

## Theming

Studio uses a **scoped palette** on `.studio-shell` (`studio-theme.css`) so the editor can be themed independently of the docs site. Override `--studio-control-surface-*` tokens to restyle every sidebar control at once.
