Studio UI is the design system for the **chart editor** ‚Äî sidebars, property rows, motion controls, and pickers. Chart rendering primitives live in `@bklitui/ui` (see the main site docs); **this app** documents editor-only UI in `packages/studio`.

Local-only ‚Äî not deployed.

| Add shadcn components | `cd packages/studio && pnpm ui:add <name>` |

**Custom** (hand-maintained): [Yes/No switch](/docs/custom-components), [Studio slider](/docs/custom-components#studio-slider), scrub `number-field`, OKLCH color picker, motion bezier editor + ease presets, fill/pattern/legend pickers.

**Icons:** all Studio UI icons come from [`@bklitui/icons`](https://www.npmjs.com/package/@central-icons-react/all) (Central Icon System). Motion presets use `IconAnimationEaseOut`, `IconAnimationEase`, `IconAnimationEaseIn`, and `IconAnimationElastic`; the bezier editor footer uses `IconBezierCurves`. See [Motion](/docs/motion).

**Shadcn-managed** (22): `button`, `dialog`, `dropdown-menu`, `input`, `popover`, `select`, `slider`, `toggle`, `toggle-group`, `tooltip`, and more ‚Äî run `pnpm dlx shadcn@latest info --json` from `packages/studio`.

## Theming

Studio uses a **scoped palette** on `.studio-shell` (`studio-theme.css`) so the editor can be themed independently of the docs site. Override `--studio-control-surface-*` tokens to restyle every sidebar control at once.
